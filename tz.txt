Для общения с внешним миром у платы должно быть 2 интерфейса, первый - это web-интерфейс, второй должен быть реализован через tcp протокол. В web-интерфейсе я хочу увидеть два текстовых поля для принятых и переданных данных, чтобы я мог прям в окошке написать что-то, нажать кнопочку и отправить. Так же мне необходимы поля для настроек, чтобы я мог считывать и передавать их (get/set). Еще необходимо выдавать отладочные графики и просто отладочные переменные, которые не являются параметрами для настройки (например дистанция, направление, количество ошибок, амплитуда принятого сигнала, медиана сигнала и т.п.). Для интерфейса через tcp протокол необходимо просто передавать и принимать данные, параметры, навигационную информацию.

У драйвера должен быть интерфейс следующий - при его загрузке в папке /dev/ должнен появляться его файл, при записи в этот файл данные будут передаваться, при чтении из него данные будут приниматься, при использовании функции ioctl можно устанавливать или считывать параметры, бинарные данные в виде графиков можно получать через mmap. Таким образом, если даже не будет web-интерфейса, то с драйвером можно будет полноценно работать даже из консоли.
